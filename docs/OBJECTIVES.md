# Section 2 — Objectives

## 2.1 System/IT Objectives

Each objective includes a specific, testable success criterion that can be verified through automated testing, manual QA, or instrumentation.

| ID | Objective | Success Criterion | Verification Method |
|----|-----------|-------------------|---------------------|
| SYS-01 | Deploy a geospatial 3D map viewer using Mapbox GL JS integrated with Three.js overlays, capable of rendering all 30 demo properties with interactive markers, clustering, and district filtering. | Initial map load time ≤ 2 seconds on a 10 Mbps connection. All 30 markers render without visual glitches. Cluster/uncluster transitions complete in < 300ms. | Lighthouse performance audit + manual stopwatch testing on throttled connection |
| SYS-02 | Implement an AI query parser using OpenAI GPT-4o-mini that converts natural language property queries (in English, Russian, and Uzbek) into structured PostGIS-compatible filters. | ≥ 85% parse accuracy on a test set of 50 sample queries spanning all three languages. Parse accuracy defined as: all extracted filter fields match the expected output exactly. | Automated pytest suite (`test_search.py`) with 50 queries and expected JSON outputs |
| SYS-03 | Compute and display comfort analytics scores (transport, shopping, education, green space, safety) for each of the 30 demo properties using real OSM and Google Places data. | All 30 properties have non-null comfort scores. Score computation completes within the nightly cron window (< 30 minutes for all properties). Data refresh latency ≤ 24 hours from last scheduled update. | Automated test verifying non-null scores + cron job execution time logging |
| SYS-04 | Build a photo-to-3D reconstruction pipeline that accepts 8–15 JPEG/PNG images and produces a GLB file viewable in the Three.js browser viewer. | 5 demo properties successfully processed. Each resulting GLB file ≤ 10 MB. 3D viewer loads model and reaches interactive state in ≤ 5 seconds. Pipeline handles API failures gracefully with retry logic and user-facing status updates. | Manual testing of 5 property uploads + file size verification + viewer load timing |
| SYS-05 | Implement a REST API with complete CRUD operations for properties, comfort scores, AI search, 3D reconstruction management, and analytics. All endpoints authenticated via API key. | API response times: p95 < 500ms for search endpoints, p95 < 200ms for property detail retrieval. All endpoints return proper HTTP status codes (200, 201, 400, 401, 404, 422, 500). Rate limiting enforced at 100 requests/minute per API key. | Automated integration tests (`test_api.py`) + load testing with `wrk` or `ab` |
| SYS-06 | Containerize the entire application stack (FastAPI backend, React frontend, PostgreSQL+PostGIS, Redis, Nginx) using Docker Compose for single-command local deployment. | `docker-compose up` starts all 5 services within 60 seconds. All services pass health checks. Frontend accessible at `http://localhost`, API at `http://localhost/api/v1/docs`. | Manual `docker-compose up` + curl health check endpoints |
| SYS-07 | Implement a CI/CD pipeline using GitHub Actions that automatically runs tests, builds Docker images, and deploys to the production server on push to the `main` branch. | Pipeline completes in ≤ 10 minutes. Failed tests block deployment. Successful deployments result in zero-downtime container restarts. | GitHub Actions run history + deployment verification |
| SYS-08 | Build an embeddable JavaScript widget that partner platforms can integrate via a single `<script>` tag, rendering the comfort panel and 3D viewer inside an iframe. | Widget renders correctly in Chrome, Firefox, Safari, and Edge. Widget loads in ≤ 3 seconds. Widget respects responsive sizing (adapts to container width from 320px to 1920px). Cross-origin messaging works for theme and property ID configuration. | Cross-browser manual testing on a dummy HTML page |

## 2.2 Business Objectives

| ID | Objective | Success Criterion | Measurement Method |
|----|-----------|-------------------|---------------------|
| BIZ-01 | Increase average property listing view time by providing immersive 3D previews and comfort analytics. | ≥ 40% increase in average time spent on property pages that display PropVision widgets, compared to baseline pages without widgets (measured during user testing). | A/B comparison during focus group testing: participants view properties with and without PropVision features, tracked via screen recording and session timers |
| BIZ-02 | Achieve high engagement with AI-powered natural language search. | ≥ 30% click-through rate from AI search results to property detail pages during user testing sessions. At least 70% of test participants successfully use the AI search to find a relevant property. | Focus group task completion metrics + event logging (search → click) |
| BIZ-03 | Secure integration partnership with at least 1 local real estate platform for post-MVP pilot deployment. | Signed letter of intent (LOI) or MOU with at least one platform (OLX Uzbekistan, Uysot, or Birbir) by the end of the 100-day project timeline. Integration guide and API documentation delivered to the partner's technical team. | Written LOI/MOU document + partner acknowledgment of receiving integration documentation |
| BIZ-04 | Validate the comfort analytics concept as a decision-support tool for property buyers. | ≥ 75% of user testing participants rate the comfort analytics dashboard as "useful" or "very useful" (on a 5-point Likert scale). At least 60% of participants reference comfort scores when explaining their property preferences. | Post-session survey (Google Form) + qualitative analysis of focus group transcripts |
| BIZ-05 | Demonstrate that the MVP can operate within a constrained budget suitable for a student/early-stage project. | Total monthly infrastructure cost ≤ $60/month (AWS Lightsail + API usage within free tiers). Third-party API costs (OpenAI, Google Places) remain within free or minimal-usage tiers during the demo period. | Monthly AWS billing report + API usage dashboard review |
